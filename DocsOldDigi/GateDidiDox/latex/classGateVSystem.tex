\doxysubsection{Detailed Description}
A \mbox{\hyperlink{classGateVSystem}{Gate\+VSystem}} is an integrated interface to a GATE geometry. 

It can read and return information on the geometry according to a predefined set-\/up


\begin{DoxyItemize}
\item \mbox{\hyperlink{classGateVSystem}{Gate\+VSystem}} -\/ by \href{mailto:Daniel.Strul@iphe.unil.ch}{\texttt{ Daniel.\+Strul@iphe.\+unil.\+ch}} (2002)
\item A system provides a pre-\/defined model of a geometry, such as a scanner or a source. This model is built as a tree of system-\/components (\mbox{\hyperlink{classGateSystemComponent}{Gate\+System\+Component}}), starting from the \textquotesingle{}tree-\/base\textquotesingle{} (m\+\_\+base\+Component). In this model, each component has a specific role (detector head, crystal matrix, collimator...)
\item System components are activated when they are connected to an inserter of the geometry. Once a component is thus connected, it can read the inserter properties, such as its dimensions, position, movement parameters, number of copies...
\item For example, a typical PET scanner would incorporate a component for the detector blocks (\textquotesingle{}rsector\textquotesingle{}, \textquotesingle{}block\textquotesingle{}, \textquotesingle{}bucket\textquotesingle{}...). This component would then be connected to the one geometry inserter that models this detector block. Once this connection is done, one can read the scanner properties that are related to the blocks\+: number of rings, number of blocks per ring, internal diameter...
\item A system is also responsible for computing output volume IDs (\mbox{\hyperlink{classGateOutputVolumeID}{Gate\+Output\+Volume\+ID}}), which are used for data analysis and image reconstruction. This task is actually delegated to the component tree.
\item To see a concrete application of this mechanism, check the class \mbox{\hyperlink{classGateCylindricalPETSystem}{Gate\+Cylindrical\+PETSystem}}
\item Note\+: from July to Oct. 2002, a system was a vector of system-\/levels. It was redesigned as a tree of system-\/components in Oct 2002.
\end{DoxyItemize}
