\doxysubsection{Detailed Description}
Framework base-\/class for all repeater classes. 


\begin{DoxyItemize}
\item \mbox{\hyperlink{classGateVGlobalPlacement}{Gate\+VGlobal\+Placement}} -\/ by \href{mailto:Daniel.Strul@iphe.unil.ch}{\texttt{ Daniel.\+Strul@iphe.\+unil.\+ch}}
\item The Gate\+VObject\+Move is derived from \mbox{\hyperlink{classGateClockDependent}{Gate\+Clock\+Dependent}}. It is responsible for computing the orientation and position of a series of copies of a volume, according to a repetition algorithm equation programmed by the user.
\item \mbox{\hyperlink{classGateVGlobalPlacement}{Gate\+VGlobal\+Placement}} is an abstract class\+: specific repeaters can be modelled by deriving application classes from it. In each application class, it is the responsability of the developper to implement a model of the repetition in the pure virtual method \mbox{\hyperlink{classGateVGlobalPlacement_afd66e610bd702eb6ad6e7374f8c16b0e}{Push\+My\+Placements()}}.
\item It is also the responsability of the developper to implement the pure virtual method \mbox{\hyperlink{classGateVGlobalPlacement_a0e8c9af31eb5d57d6fa92d2fe2f86a4b}{Describe\+Myself()}}. This method should print-\/out a description of the repeater (i.\+e. its parameters). 
\end{DoxyItemize}

If the repeater is enabled, it repeats each placement in the input queue using the user-\/defined virtual method Push\+My\+Placements. The series thus obtained are all placed into the repeater placement queue, which is returned to the caller. If the repeater is disabled, it directly places a copy of each input placement into the output queue, so that the output queue is a direct copy of the input queue.
