\doxysubsection{Detailed Description}
Abstract base-\/class for pulse-\/processor components of the digitizer. 


\begin{DoxyItemize}
\item \mbox{\hyperlink{classGateVPulseProcessor}{Gate\+VPulse\+Processor}} -\/ by \href{mailto:Daniel.Strul@iphe.unil.ch}{\texttt{ Daniel.\+Strul@iphe.\+unil.\+ch}}
\item \mbox{\hyperlink{classGateVPulseProcessor}{Gate\+VPulse\+Processor}} is the abstract base-\/class for all pulse-\/processors. The pulse-\/processors are pluggable modules that can be inserted into the digitizer module\textquotesingle{}s list.
\item Their role is to process a list of pulses (coming from the hit-\/convertor or from another pulse-\/processor) and to return a new, processed list of pulses, which can be fed to another pulse-\/processor or to the coincidence sorter for example.
\item When developping a new pulse-\/processor, one should\+:
\begin{DoxyItemize}
\item develop the pulse-\/processing functions (see below);
\item develop a messenger for this pulse-\/processor;
\item add the new pulse-\/processor to the list of choices available in \mbox{\hyperlink{classGatePulseProcessorChainMessenger}{Gate\+Pulse\+Processor\+Chain\+Messenger}}.
\end{DoxyItemize}
\item To develop the pulse-\/processing functions, the developper has two options\+:
\begin{DoxyItemize}
\item The easiest option is to re-\/use the pre-\/defined method \mbox{\hyperlink{classGateVPulseProcessor_adaa414f484d8b3f67f2297ddbc2f6cf7}{Process\+Pulse\+List()}}. This method processes an input pulse-\/list by repeatedly calling the method \mbox{\hyperlink{classGateVPulseProcessor_a228e213fe778039eac929cb83479fc72}{Process\+One\+Pulse()}} once for each input-\/pulse. The responsability of the developper then is to implement the pure virtual method \mbox{\hyperlink{classGateVPulseProcessor_a228e213fe778039eac929cb83479fc72}{Process\+One\+Pulse()}} so as to perform the required processing.
\item The other option is to overload the method \mbox{\hyperlink{classGateVPulseProcessor_adaa414f484d8b3f67f2297ddbc2f6cf7}{Process\+Pulse\+List()}} (if the pulse-\/processing sequential mechanism provided by \mbox{\hyperlink{classGateVPulseProcessor_adaa414f484d8b3f67f2297ddbc2f6cf7}{Process\+Pulse\+List()}} is not appropriate. In that case, one should provide some dummy implementation (such as \{;\}) for \mbox{\hyperlink{classGateVPulseProcessor_a228e213fe778039eac929cb83479fc72}{Process\+One\+Pulse()}}
\end{DoxyItemize}
